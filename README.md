# Market Overlay

주요 시장 지표와 개인 포트폴리오 상태를 데스크톱 오버레이로 확인하는 Windows용 투자 보조 도구입니다.

## 실행 방법

브라우저 예시는 `index.html`을 열면 바로 실행됩니다.

```powershell
Start-Process .\index.html
```

실제 오버레이처럼 항상 위에 뜨는 데스크톱 예시는 아래처럼 실행합니다. 현재 기본 실행 대상은 PySide6 버전입니다. `pythonw.exe`를 쓰면 PowerShell 콘솔 창 없이 오버레이만 뜹니다.

```powershell
& 'C:\Users\User\AppData\Local\Programs\Python\Python312\pythonw.exe' .\run_overlay.pyw
```

V3 같은 보안 프로그램이 전역 단축키나 클릭 통과를 차단하면 안전 모드로 실행할 수 있습니다. 안전 모드는 실데이터 조회와 화면 표시만 하고, 잠금/클릭통과 기능은 끕니다.

```powershell
& 'C:\Users\User\AppData\Local\Programs\Python\Python312\pythonw.exe' .\safe_overlay.pyw
```

PySide6 버전은 카드 배경만 반투명하게 만들고 글씨는 선명하게 유지합니다. 잠금 상태에서는 Qt의 입력 투명 플래그를 사용해 뒤쪽 창 클릭을 통과시킵니다.

데이터 호출만 확인하려면 콘솔에서 아래 명령을 사용합니다.

```powershell
& 'C:\Users\User\AppData\Local\Programs\Python\Python312\python.exe' .\desktop_overlay.py --check-data
```

## 현재 버전

- 현재 정식 릴리즈는 `v2.2`입니다.
- 사용자 문서 바로가기: [텍스트 사용자 가이드](USER_GUIDE_V2_2.txt), [이미지로 보는 사용자 가이드](docs/USER_GUIDE_SCREENSHOT_V2_2.md), [v2.2 패치노트](PATCH_NOTES_V2_2.txt)
- 대표 화면: [종합 오버레이 예시](docs/images/user_guide_v2_0/overlay_combined.png)
- 데스크톱 오버레이는 Yahoo Finance chart API로 실제 지수/VIX 일봉 데이터를 가져옵니다.
- 공포탐욕은 CNN dataviz JSON을 우선 사용하고, 실패하면 `Local F&G` 자체 계산값으로 대체 표시합니다.
- 외부 데이터 호출 실패 시 화면에 실패 상태를 표시하고, 정상값이 있으면 마지막 정상 갱신값을 유지하며 없으면 수치 카드에 `--`를 표시합니다.
- 오버레이 본체는 밝은 반투명 카드형 HUD이고, 조작은 별도 컨트롤 창에서 합니다.
- 설정 탭에서 오버레이 화면을 `시장`, `종합`, `내자산` 중 선택합니다.
- 시장 오버레이는 시장 상태, USD/KRW, F&G, VIX, S&P 500, Nasdaq Composite, KOSPI 등을 표시합니다.
- 종합 오버레이는 시장요약과 포트폴리오 핵심 수치를 폭이 좁은 HUD에 함께 압축 표시합니다.
- 내 포트폴리오 오버레이는 손익률 바, 포트 비중, 투자/현금 비중, 총평가액/원금/수익/전일대비 카드를 표시합니다.
- 상단에는 시장 상태 라벨과 USD/KRW 환율 카드가 표시됩니다.
- 컨트롤 창은 작업표시줄에 남기 때문에 이동, 잠금, 새로고침, 종료를 거기서 처리할 수 있습니다.
- 시장 탭의 `오버레이 숨기기/보이기` 버튼으로 HUD 표시 여부를 토글합니다.
- 시장 탭에는 HUD와 같은 핵심 지표 요약 카드와 선택한 지수 카드가 항상 표시됩니다.
- Nasdaq Composite, S&P 500, KOSPI의 이동평균과 괴리율을 계산합니다.
- 포트폴리오 탭에서 Yahoo Finance 심볼을 입력해 종목별 현재가, 전일 대비, 이동평균 괴리율을 볼 수 있습니다.
- 포트폴리오 탭의 `종합/포트명` 필터 탭은 편집 입력부 위에 있으며, 선택에 따라 편집 목록, 요약 카드, 평가 목록을 함께 바꿉니다.
- 종합 결과 목록은 `포트별/종목별` 롤업으로 전환할 수 있고, 펼침 행에서 포트 또는 종목의 상세 보유를 확인합니다.
- v1.5부터 종합 목록은 예수금을 포함한 전체 순자산을 기준으로 포트별 순자산 비중과 종목별 평가액 비중을 표시합니다.
- 편집 목록과 평가 목록은 종목 수와 무관하게 정해진 화면 영역을 유지하며, 추가 종목은 목록 내부 스크롤로 확인합니다.
- 포트폴리오 탭은 보유수량을 기준으로 총 평가액, 전일대비 금액과 수익률, 원화 환산 평가액, 보유비중, 평단 원금 대비 종목별 수익률을 계산합니다.
- v1.3부터 총 평가액은 보유자산 평가액 기준으로 표시하고, 예수금 포함 순자산은 보조 문구로 분리합니다.
- 포트폴리오 요약은 총 평가액, 원금, 수익, 전일대비, 현금비중의 5개 카드로 표시합니다.
- 총 평가액은 원금 대비 평가손익 방향으로 색을 표시하고, 수익 카드에는 평가손익, 실현수익, 배당금을 합산한 금액과 원금 대비 수익률을 함께 표시합니다.
- 전일대비 카드에는 전일대비 금액과 수익률을 함께 표시합니다.
- 종합의 포트별/종목별 목록과 펼친 행은 비중 내림차순으로 표시하며, 포트 선택 탭은 고정 순서를 유지합니다.
- 포지션은 추가, 매수, 매도, 수정, 삭제 방식으로 조정할 수 있습니다. 보유 종목을 선택하면 입력창에 보유수량과 평단가가 불러와집니다.
- 종목 입력은 현재 종목명 데이터에 있는 티커/국내 코드를 자동완성하며, 선택된 보유종목 행을 다시 클릭하면 선택이 해제됩니다.
- 선택되지 않은 상태의 `추가`는 새 보유항목을 등록하고, 목록 선택 후 `수정`은 현금 변동 없이 보유수량, 평단가, 표시명, 보유 시작일을 수정합니다.
- 중복 보유는 포트폴리오와 정규화된 종목코드를 함께 기준으로 판단하므로 같은 종목을 서로 다른 포트에는 등록할 수 있습니다.
- 매수/매도는 별도 거래수량 칸을 사용합니다. 보유수량 수정과 거래 입력은 서로 분리됩니다.
- 보유 시작일은 `yyyymmdd` 형식으로 입력하며, 비워두고 수정하면 저장일이 자동 입력됩니다.
- 미국 종목 평단가는 기본 KRW이며 USD/KRW 중 선택할 수 있고, 국내 종목은 KRW로 고정됩니다.
- 선택된 포지션은 `삭제` 버튼으로 제거할 수 있습니다.
- 포트폴리오 이름은 `종합 | 포트명 | +` 탭 줄에서 추가하고, 활성 탭 더블클릭으로 변경하며, hover `x`와 확인창으로 삭제합니다. 삭제 아이콘 공간은 항상 예약해 hover 전후 탭 폭이 움직이지 않습니다.
- 포트폴리오 이름을 변경해도 내부 포트 ID로 수익과 배당 이력 연결이 유지됩니다.
- 포트폴리오별 예수금은 KRW와 USD를 각각 저장할 수 있고, 조회 화면에서는 원화 환산으로 자동 합산합니다.
- 매수/매도 거래가는 비어 있으면 조회된 현재가를 자동 입력하고, 사용자가 직접 수정할 수 있습니다.
- 매수 시 상장 시장 통화 기준 예수금이 줄고 매도 시 늘어납니다. 미국 종목은 USD, 국내 종목은 KRW 예수금에 반영됩니다.
- 매수 시 예수금이 부족해도 거래 입력은 허용하며, 음수 예수금은 현금비중 카드에서 경고색으로 표시합니다.
- 포트폴리오 탭의 입력부는 포트 관리, 조회 포트 탭, 종목 관리, 거래/예수금 행으로 나누어 표시합니다.
- 포트폴리오 탭의 카드 아래 영역은 좌우로 나뉘며, 왼쪽 `비중/트리맵` 시각화와 오른쪽 보유 목록 폭을 드래그로 조절할 수 있습니다.
- `비중/트리맵` 시각화 영역은 `기본값` 버튼으로 기본 폭으로 되돌릴 수 있고, 조절한 폭은 다음 실행 때 복원됩니다.
- 매매기록 탭은 매수/매도 내역을 원장으로 관리하며, 포트/종목/기간/구분/통화/상태 필터와 헤더 정렬을 지원합니다.
- 매매기록 탭은 매수단가, 매도단가, 수량, 1주당 손익, 총손익, 수익률 중심으로 표시합니다.
- 매매기록 수동 추가는 현재 보유종목 자동완성 입력을 기준으로 행을 만듭니다.
- 수동 매매기록과 수동 배당은 `작성중` 상태로 시작하며 `반영`하기 전에는 포트 합계 계산에서 제외됩니다.
- 배당 탭에서 Yahoo Finance 배당 이벤트를 불러와 현재 보유수량 기준 배당 기록을 추가할 수 있습니다.
- v1.3부터 신규 배당만과 범위 불러오기를 분리했고 보유 시작일 이후/최근 1년/최근 3년/직접 날짜/전체 범위를 선택할 수 있습니다.
- 배당 조회 대상은 전체 보유종목 또는 특정 보유종목으로 선택할 수 있습니다.
- 배당 수동 추가도 선택한 보유종목을 기준으로 행을 만듭니다.
- 사용자 상태 및 종목명 JSON은 자동 백업 파일을 유지하며 읽기 실패 시 백업을 사용합니다.
- 최근 1년/최근 3년은 날짜가 자동 입력되고, 직접 날짜 지정은 `yyyymmdd` 형식을 사용합니다.
- 수익/배당 이력은 표에서 직접 수정할 수 있습니다.
- 미국 종목은 달러 가격과 원화 환산 금액을 함께 표시하고, 국내 종목은 원화 기준으로 표시합니다.
- 종목명은 가능한 경우 대중적인 한국어 표기명이나 Yahoo Finance 종목명을 사용합니다. 표시명 매핑은 `data\symbol_names.json`에 보관합니다.
- v1.6부터 배포본 내장 데이터와 사용자 보정을 사용하며, 사용자 표시명 보정이 항상 우선합니다. 원격 데이터팩 URL 업데이트 기능은 제거했습니다.
- v1.2부터 NASDAQ Trader 공식 심볼 디렉터리와 KIND/KRX 상장법인 목록 기준으로 표시명 매핑을 대량 확장했습니다.
- v1.4부터 KRX 상장 증권상품 검색 목록을 사용해 국내 ETF와 ETN 표시명도 내장 데이터에 포함합니다.
- 주요 미국 종목/ETF는 한국어 별칭을 우선 사용하고, 그 외 미국 티커는 공식 영문 종목명을 사용합니다.
- 공식/내부 매핑과 조회명이 모두 없는 신규/마이너 종목은 마지막 예외처리로 `개잡주`가 표시될 수 있으며, 필요하면 추가 시 표시명을 직접 입력할 수 있습니다.
- 내장 매핑에 등록된 국내 종목/ETF/ETN 단축코드는 시장 suffix를 자동 적용합니다. 예: `005930` -> `005930.KS`, `446770` -> `446770.KS`
- v1.3부터 `.KS` 조회 실패 시 `.KQ`, `.KN` 후보도 순차 재시도합니다.
- 설정 탭에서 오버레이에 표시할 지표와 이동평균 기준을 직접 선택할 수 있습니다.
- 이동평균 기준은 200일이 기본이며, 5일, 20일, 60일, 120일, 200일 중에서 고를 수 있습니다.
- 설정 탭에서 60거래일 스파크라인 표시 여부를 토글할 수 있습니다.
- 기본 지표는 Nasdaq Composite, S&P 500, KOSPI이며 추가 후보는 SOX, Russell 2000, Dow Transports, 미국 10년물 금리, 달러지수, KOSDAQ입니다.
- 상승/양호 쪽은 적색, 하락/위험 쪽은 청색 계통으로 표시합니다.
- VIX는 상승할수록 위험 신호로 보아 청색 계통으로 표시합니다.
- 설정 탭에서 시장 지표의 자동 갱신 기준 주기를 5분 또는 10분으로 선택할 수 있습니다.
- 설정 탭의 `로그 폴더 열기` 버튼으로 진단 로그 폴더를 열 수 있습니다. 로그는 네트워크/파싱/state/worker 문제를 추적하기 위한 것이며 장부 세부값은 남기지 않습니다.
- 보유종목 전체 현재가는 시장 주기의 1/2/3/6배 또는 수동만으로 설정하며, 고유 보유종목 수에 맞춘 권장 안내를 표시합니다.
- 시장 탭의 전체 새로고침은 시장 지표와 보유종목 현재가를 즉시 함께 갱신합니다. 배당 이력은 자동 갱신하지 않습니다.
- 한 갱신 회차에서는 동일 Yahoo 심볼 응답을 재사용해 지수/환율/포트폴리오 조회의 중복 요청을 줄입니다.
- 보유종목 자동 현재가 조회는 현재 등록된 보유심볼만 대상으로 하며, 종목명 매핑 전체를 가격 조회하지 않습니다.
- `Ctrl + Alt + M`으로 컨트롤 잠금/해제를 전환합니다.
- 잠금 상태에서는 오버레이가 클릭을 통과시켜 뒤쪽 창을 그대로 조작할 수 있습니다.
- 잠금 상태에서는 작은 `잠김 / 해제` 핸들이 따로 떠서 단축키가 막혀도 마우스로 풀 수 있습니다.
- 보안 프로그램이 잠금 기능을 차단하면 잠금을 적용하지 않고 안전모드 안내를 표시합니다.

## 설치와 배포

개발 PC에서 필요한 패키지는 아래처럼 설치합니다.

```powershell
& 'C:\Users\User\AppData\Local\Programs\Python\Python312\Scripts\pip.exe' install -r .\requirements.txt
```

다른 사람에게 배포할 때 `run_overlay.pyw`만 전달하면 안 됩니다. 개발용 실행 파일은 주변 Python 파일과 PySide6 설치 환경에 의존합니다.

실행 파일 폴더는 PyInstaller로 만들 수 있습니다.

```powershell
.\build_pyside_exe.ps1
```

빌드 결과는 아래 경로에 생성됩니다.

```text
dist\MarketOverlay\MarketOverlay.exe
```

처음 배포할 때는 `dist\MarketOverlay` 폴더 전체를 압축해서 전달하는 방식이 가장 단순합니다. 단일 exe가 필요하면 PyInstaller의 `--onefile` 옵션을 추가할 수 있지만, PySide6 앱은 시작 시간이 느려지고 보안 프로그램 오탐 가능성이 더 커질 수 있습니다.

일반 사용자에게는 설치 마법사 형태가 더 편합니다. Inno Setup 6를 설치한 뒤 아래 명령을 실행하면 PyInstaller 빌드와 설치 파일 생성을 한 번에 처리합니다.

```powershell
.\build_installer.ps1
```

설치 파일은 아래 경로에 생성됩니다.

```text
release\v2.2\MarketOverlaySetup_v2.2.exe
```

설치 빌드는 `build\installer_dist\MarketOverlay`의 staging 실행 파일을 사용하므로 `dist\MarketOverlay` portable 앱이 실행 중이어도 최신 설치 파일을 만들 수 있습니다. portable 실행 파일 자체를 교체하려면 실행 중인 앱을 닫고 `build_pyside_exe.ps1`을 실행합니다.

설치 파일은 사용자 권한으로 `{localappdata}\Programs\Market Overlay`에 설치되도록 설정했습니다. 안내 문서 `USER_GUIDE_V2_2.txt`와 이번 변경 이력 `PATCH_NOTES_V2_2.txt`도 설치 폴더에 함께 포함됩니다. 오너용 누적 이력은 `OWNER_PATCH_HISTORY_V2_2.txt`로 관리합니다.

소스 수정까지 허용하고 싶으면 private source repo 접근 권한을 부여합니다. 일반 사용자에게는 public distribution repo의 GitHub Release 설치파일과 최신 안내/패치노트를 전달합니다. 이전 산출물은 버전별 보관 경로에 유지합니다.

GitHub 운영 구조:

- private source repo: `https://github.com/Link-1214/market-overlay-private`
- public distribution repo: `https://github.com/Link-1214/market-overlay`
- public repo에는 설치파일, 사용자 가이드, 패치노트, 릴리즈 노트만 둡니다.
- GitHub가 자동으로 표시하는 `Source code(zip/tar.gz)`는 public 배포 태그 기준 압축파일이며 실제 Python 소스가 들어가지 않게 관리합니다.

설치 파일은 같은 AppId와 설치 경로를 사용하므로 기존 Market Overlay를 별도 프로그램으로 병존시키지 않고 덮어씁니다. 사용자 포트폴리오 데이터는 `%APPDATA%\MarketOverlay`에 보관되어 업데이트 후에도 유지되며, 제거 시 개인 데이터 삭제 여부를 별도로 선택합니다.

버전 규칙:

- 첫 번째 소수점: 버그 수정, 빌드 고도화, 기능 수정이 있을 때 올립니다. 예: `1.1` -> `1.2`
- 두 번째 소수점: 표시명 매핑 같은 단순 데이터 최신화만 있을 때 올립니다. 예: `1.2` -> `1.2.1`
- `1.9` 다음 기능/수정 버전은 `2.0`으로 올립니다.

앱 설정과 포트폴리오 데이터는 설치 폴더가 아니라 사용자 AppData 아래에 저장됩니다.

```text
%APPDATA%\MarketOverlay\market_overlay_state.json
```

## 계산식

```text
전일 대비 등락률 = (현재가 / 전일 종가 - 1) * 100
이동평균 괴리율 = (현재가 / 선택한 이동평균 - 1) * 100
```

시장 상태 라벨은 `극단적 탐욕`, `탐욕`, `중립`, `공포`, `극단적 공포` 5단계입니다. CNN F&G, IXIC/SPX 이동평균 괴리율, VIX 절대 레벨을 함께 보며, 현재 정책은 F&G 40%, IXIC/SPX MA 괴리율 35%, VIX 25% 가중 점수로 산출합니다.

```text
CNN F&G 자체 구분:
극단적 공포: 0~24
공포: 25~44
중립: 45~55
탐욕: 56~75
극단적 탐욕: 76~100

시장 상태:
F&G, IXIC/SPX MA, VIX를 각각 점수화한 뒤 가중 평균으로 최종 라벨을 정합니다.
```

한국장 심리 대체 지표로는 VKOSPI가 가장 성격이 가깝지만, 현재 앱의 기본 데이터 경로에서는 안정적으로 연결하지 않았습니다. 그래서 국장은 우선 KOSPI/KOSDAQ의 이동평균 괴리와 전일 흐름을 중심으로 봅니다.

`Local F&G`는 CNN 지수와 동일한 공식값이 아니라, CNN 데이터 접근이 실패할 때 공개 데이터로 계산하는 대체 지표입니다. 현재 버전은 2020년 이후 CNN 히스토리 데이터에 맞춰 선형 보정을 적용합니다.

```text
Local F&G raw =
  S&P 500 125일 모멘텀
+ S&P 500 20일 모멘텀
+ Nasdaq 100 / S&P 500 상대강도
+ VIX 1년 분위
+ VIX 20일 변화
+ SPY / IEF 안전자산 수요
+ HYG / LQD 정크본드 수요

Local F&G calibrated = raw * 0.7949534329 + 3.2265061011
```
