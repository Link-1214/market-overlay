Market Overlay v1.6 사용자 가이드

1. 프로그램 성격과 실행

Market Overlay는 시장 지표와 개인 포트폴리오 흐름을 빠르게 확인하는 보조 도구입니다.
실시간 체결 프로그램이나 매수ㆍ매도 판단을 대신하는 서비스가 아닙니다.
설치 후 시작 메뉴 또는 선택한 바탕화면 바로가기에서 `Market Overlay`를 실행합니다.

2. 시장 탭과 오버레이

- `오버레이 잠금`: HUD를 항상 위에 두되 클릭이 뒤 창으로 통과하도록 합니다.
- `오버레이 숨기기/보이기`: HUD만 감추거나 다시 표시합니다.
- `전체 새로고침`: 시장 지표와 보유종목 현재가를 함께 즉시 갱신합니다.
- 상승 방향은 적색, 하락ㆍ위험 방향은 청색 계통으로 표시합니다.
- Local F&G는 CNN 공식 지수가 아니라 Yahoo 기반 자료로 계산한 로컬 보조값입니다.

데이터 표시 주의:
- 정상 조회 시 데이터 출처가 표시됩니다.
- 조회 실패 후 이전 정상값이 있으면 `마지막 정상값 표시 중`으로 알리고 이전값을 유지합니다.
- 정상값이 없는 상태에서 실패하면 실제 수치 카드는 `--`로 숨기고 조회 실패 상태를 알립니다.

3. 포트폴리오 탭

포트 선택 및 관리:
- `종합`은 전체 보유자산을 봅니다.
- 포트명 탭을 누르면 해당 포트의 종목 입력, 예수금, 결과가 함께 전환됩니다.
- 탭 줄 마지막의 `+`를 누르고 이름을 입력한 뒤 다른 영역을 클릭하거나 Enter를 누르면 포트가 추가됩니다. Esc 또는 빈 이름으로 벗어나기는 취소입니다.
- 선택된 포트 탭을 더블클릭하면 이름을 수정할 수 있으며, 다른 영역을 클릭하거나 Enter를 누르면 저장됩니다.
- 포트 탭에 마우스를 올리면 표시되는 `x`로 삭제할 수 있습니다. 삭제 확인창이 먼저 뜹니다.
- 삭제하면 해당 포트의 현재 보유종목과 예수금은 제거되지만, 과거 수익/배당 이력은 기록 목적으로 보존됩니다.

보유종목 편집:
- 종목코드 또는 이름을 입력해 자동완성에서 선택합니다. 국내 ETF/ETN도 지원되는 내장 목록에서 검색됩니다.
- 선택이 없는 상태의 `추가`는 새 보유항목을 등록하며 예수금은 변하지 않습니다.
- 목록의 종목을 선택하면 `수정`으로 현재 수량, 평단가, 표시명, 보유 시작일을 고칠 수 있습니다.
- 같은 행을 다시 누르면 선택이 풀립니다.
- `매수`와 `매도`는 거래수량ㆍ거래가를 사용하며 해당 포트의 예수금도 함께 반영합니다.
- 기존 보유종목을 추가 매수하면 수량과 평단가가 가중평균으로 갱신되고 기존 보유 시작일은 유지됩니다.
- 보유 시작일은 `yyyymmdd` 형식으로 입력합니다. 비워 새 항목을 저장하면 저장일을 사용합니다.
- 미국 종목 평단가는 KRW 또는 USD를 선택할 수 있고 국내 종목은 KRW입니다.

요약과 종합 보기:
- 카드에는 총 평가액, 원금, 수익과 원금대비 수익률, 전일대비, 현금비중을 표시합니다.
- `종합`의 `포트별`은 포트 단위 순자산을, `종목별`은 여러 포트에 걸친 같은 종목을 합산해 보여줍니다.
- 두 목록 모두 전체 순자산 대비 `비중`이 큰 순으로 표시되고, 행을 펼쳐 상세 항목을 볼 수 있습니다.
- 상세 종목 목록은 열 너비를 고정해 긴 종목명이 뒤 열을 밀지 않으며, `수익률` 열에서 평단 원금 대비 평가 손익률을 확인합니다.

4. 수익 탭

- 포트폴리오 탭에서 매도한 거래는 실현수익 기록으로 자동 남습니다.
- 수동 추가 기록은 먼저 `작성중` 상태로 저장되며 `반영`으로 바꾼 뒤에만 요약 계산에 포함됩니다.
- 미국 종목의 실현수익은 USD 금액과 원화 환산 금액을 함께 관리합니다.
- v1.5에서 기록한 USD 수익이 있다면 v1.6 설치 후 원화환산액이 올바른지 확인합니다.

5. 배당 탭

- `신규 배당만`은 각 포트ㆍ종목별로 저장된 마지막 배당일 이후 기록만 추가합니다.
- `범위 불러오기`는 보유 시작일 이후, 최근 1년, 최근 3년, 직접 날짜, 전체 중 선택합니다.
- 조회 종목은 전체 보유종목 또는 특정 보유종목으로 고릅니다.
- 수동 추가 배당은 `작성중`으로 시작하고, 실제 수령 내용을 확인한 후 `반영`합니다.
- 배당은 현재 입력된 보유수량 기준으로 계산하므로 과거 배당 기준일의 실제 수량과 다를 수 있습니다.

6. 설정과 자동 갱신

- 이동평균 기준은 5, 20, 60, 120, 200일 중 선택하며 기본은 200일입니다.
- 스파크라인과 오버레이 표시 지표를 선택할 수 있습니다.
- 시장 지표 갱신 기준 주기는 5분 또는 10분입니다.
- 보유종목 현재가는 시장 주기의 1배, 2배, 3배, 6배 또는 수동만으로 정합니다.
- 전체 새로고침은 자동 설정과 관계없이 시장과 보유종목을 함께 갱신합니다. 배당 이력은 자동 조회하지 않습니다.
- 종목명은 앱 배포본의 내장 목록을 기본으로 사용하고, 사용자가 입력한 표시명 보정은 계속 우선합니다.

7. 백업, 업데이트, 제거

설정 탭의 `백업 내보내기`로 개인 장부와 사용자 표시명 보정을 파일로 보관할 수 있습니다.
`백업 가져오기`는 현재 상태를 자동 백업한 다음 선택 파일 전체를 복원합니다.
가져오기 중 오류가 나면 일부 데이터만 반영된 상태를 남기지 않고 기존 상태로 되돌립니다.

사용자 데이터 저장 위치:
`%APPDATA%\MarketOverlay`

주요 파일:
- `market_overlay_state.json`: 포트폴리오, 수익, 배당, 설정
- `market_overlay_state.json.bak`: 직전 정상 저장 백업
- `symbol_names_user.json`: 사용자가 고친 표시명

업데이트 설치는 개인 데이터를 삭제하지 않습니다.
제거 프로그램을 실행하면 개인 데이터까지 삭제할지 묻습니다. 보존을 선택하면 재설치 후 다시 사용할 수 있습니다.

8. 주의 사항

- Yahoo Finance 기반 조회는 지연, 누락 또는 일시 실패가 있을 수 있습니다.
- Local F&G와 시장 상태 라벨은 참고용이며 증권사 수치나 공식 CNN Fear & Greed와 같지 않습니다.
- 배율이 다른 듀얼 모니터에서는 운영체제 배율에 따라 HUD의 물리적 크기가 달라 보일 수 있습니다. v1.6은 이동ㆍ잠금 시 위치 이탈을 보정하지만 모니터 배율 자체를 강제로 같게 만들지는 않습니다.
- 중요한 장부 수정 전에는 백업을 내보내 두는 것이 좋습니다.
