Market Overlay v1.6.0 Patch Notes
작성일: 2026-05-26

v1.5 -> v1.6 변경 사항

1. 장부 데이터 보호 수정
- USD 실현수익 기록은 유효한 환율을 받기 전 기존 원화환산액을 0원으로 덮어쓰지 않도록 수정했습니다.
- 같은 날 같은 가격과 수량으로 여러 번 매도해도 각각의 실현수익 기록이 남습니다.
- 기존 종목을 추가 매수하거나 일부 매도할 때 보유 시작일이 거래일로 바뀌지 않습니다.
- 저장 실패를 화면에 알리고, 기존 정상 JSON 백업을 유지합니다.
- 매도 거래의 수익ㆍ예수금ㆍ보유수량은 하나의 저장 단위로 기록하여 저장 오류 때 일부만 남지 않게 했습니다.
- 첫 실행의 예제 보유종목을 제거하여 신규 사용자는 빈 장부로 시작합니다.

2. 시장 데이터 실패 표시 개선
- 시장 데이터 조회 실패 시 이전 정상값이 없으면 카드 숫자를 `--`로 숨기고 실패 상태를 알립니다.
- 세션 중 정상 데이터를 한 번 받은 뒤 조회에 실패하면 데모값 대신 마지막 정상 갱신값을 표시하고 실패 상태를 알립니다.

3. 포트폴리오 조작 간소화
- 상단 관리 버튼 줄을 없애고 `종합 | 포트명 | +` 탭 줄에서 포트폴리오를 관리하도록 변경했습니다.
- `+` 탭은 새 포트 추가, 활성 포트 탭 더블클릭은 이름 변경, 탭의 `x`는 삭제 확인창을 엽니다.
- 탭의 삭제 버튼 영역을 고정해 마우스 오버 때 탭 폭이 움직이지 않게 하고, 이름 변경은 더블클릭 진입 후 다른 영역 클릭으로도 저장되도록 다듬었습니다.
- 포트 삭제 시 보유종목과 예수금은 제거되지만 과거 수익/배당 기록은 보존된다는 정책을 확인창에 표시합니다.
- `종합` 상태에서는 대상이 불명확한 포트 편집 입력을 비활성화합니다.
- 보유종목 상세 목록의 열 배분을 고정하고, 비중 옆에 평단 원금 대비 종목별 `수익률` 열을 추가했습니다.

4. 백업ㆍ설치ㆍ데이터 운영
- 설정 탭에 개인 데이터 `백업 내보내기`와 `백업 가져오기`를 추가했습니다.
- 가져오기 전 현재 장부를 자동 백업하고, v1.6 백업 포맷은 이후 상위 버전에서 읽을 수 있도록 스키마 번호를 포함합니다.
- 백업 가져오기 도중 오류가 발생하면 부분 반영 파일을 남기지 않고 가져오기 전 상태로 되돌립니다.
- 저장 경로를 실제 `%APPDATA%\MarketOverlay` 기준으로 정리하고, 기존 v1.5 고정 경로 데이터도 읽어 이전할 수 있게 했습니다.
- 원격 종목명 데이터팩 업데이트 기능은 제거했습니다. 내장 종목명은 배포본에서 갱신하며 사용자 직접 표시명 보정은 계속 우선합니다.
- 언인스톨 시 개인 장부 데이터까지 삭제할지 별도로 묻고, 기본 선택은 보존입니다.
- 업그레이드 설치 시 이전 PyInstaller 내부 런타임 폴더를 정리한 후 신규 번들을 설치합니다.

5. 화면 안정성
- 서로 배율이 다른 모니터 사이에서 오버레이를 옮기거나 잠금/해제를 반복할 때 창이 화면 밖으로 벗어나거나 위치가 튀는 문제를 줄이도록 보정했습니다.

기존 v1.5 사용자 확인 사항

- v1.5에서 USD 실현수익을 기록한 사용자는 v1.6 실행 후 해당 기록의 원화환산액을 확인해 주세요. 이미 0원으로 저장된 기록은 원래 환산값을 자동 복원할 근거가 없습니다.
- v1.5에서 네트워크 조회 실패 중 표시된 시장 숫자는 실제 조회값이 아닌 데모값일 수 있습니다.
