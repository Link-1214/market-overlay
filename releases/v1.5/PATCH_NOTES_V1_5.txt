Market Overlay v1.5.0 Patch Notes
작성일: 2026-05-25

v1.4 -> v1.5 변경 사항

1. 종합 화면 자산 비중 표시 추가
- `포트별` 목록에 각 포트의 전체 총자산 대비 비중을 `비중`으로 간결하게 표시합니다.
- 포트별 비중은 해당 포트의 보유 평가액과 예수금을 합산한 순자산 기준입니다.
- `종목별` 목록과 펼친 상세 종목 행에는 각 종목 평가액의 전체 총자산 대비 비중을 표시합니다.
- 전체 총자산 분모에는 보유 평가액과 예수금이 함께 포함됩니다.

2. 종합 롤업 가독성 보완
- 보유 종목도 예수금도 없는 빈 포트는 종합 포트별 목록에서 표시하지 않습니다.
- 종합 목록과 펼친 상세 표의 비중 표기는 `비중`으로 간결하게 통일합니다.
- `포트별`, `종목별`, 펼친 상세 행은 비중이 큰 자산부터 내림차순으로 표시합니다.
- 상단 포트 선택 탭은 갱신으로 위치가 바뀌지 않도록 기존 순서를 유지합니다.

3. 용어 표기 정리
- 화면의 `전일`, `전일비` 표기를 `전일대비`로 통일했습니다.

4. 자동 갱신 정책과 조회 최적화
- 갱신 주기 설정을 `설정` 탭으로 이동하고, 시장 지표의 5분/10분 기준 주기를 선택할 수 있게 했습니다.
- 보유종목 전체 현재가는 시장 기준 주기의 1배, 2배, 3배, 6배 또는 수동만으로 선택할 수 있습니다.
- 시장 탭의 `전체 새로고침`은 자동 정책과 관계없이 시장 지표와 보유종목을 함께 즉시 갱신합니다.
- 설정 탭에 고유 보유종목 수에 따른 보유종목 갱신 배수 권장 안내를 표시합니다.
- 한 갱신 회차 안에서 지수, 로컬 공탐 구성 자료, 환율, 동일 보유심볼의 Yahoo 차트 응답을 재사용해 중복 요청을 줄였습니다.
- 정기 현재가 조회는 등록된 보유종목만 대상으로 하며, 종목명 데이터 전체를 주기적으로 가격 조회하지 않습니다.
- 느린 시장 조회 중 다음 주기가 도착해도 실행되지 않은 회차를 보유종목 배수 계산에 포함하지 않도록 보정했습니다.
- 추가, 수정, 삭제, 매수, 매도 중 기존 보유조회가 진행 중이면 완료 뒤 최신 보유상태 조회를 예약해 화면 수치가 뒤처지지 않게 했습니다.
- 배당 이력 불러오기는 자동 갱신 대상에서 제외하고 기존처럼 수동 실행으로 유지합니다.

5. 회귀 검증 대상
- 포트 이름 변경 후 내부 ID로 수익/배당 연결이 유지되는 동작
- 작성중 수익/배당이 계산에서 제외되는 동작
- 배당 선택 체크박스 유지 및 JSON 자동 백업 복구 동작
- 전체 새로고침, 보유종목 배수 갱신, 동일 심볼 캐시 동작
