Market Overlay v1.3.0 Patch Notes
작성일: 2026-05-25

v1.2 -> v1.3 핵심 변경

1. 포트폴리오 수치 계산 정리
- 총 평가액은 보유자산 평가액 기준으로 표시합니다.
- 예수금은 현금비중 카드와 총 평가액 보조 문구로 분리했습니다.
- 음수 예수금이 총 평가액, 원금, 전일비 수익률, 현금비중을 왜곡하던 문제를 줄였습니다.
- 수익률은 평가손익, 실현손익, 배당금을 원금 대비로 반영합니다.

2. 보유 수정과 거래 입력 분리
- 보유수량 입력칸과 매수/매도용 거래수량 입력칸을 분리했습니다.
- 수정 버튼은 예수금 변동 없이 보유수량, 평단가, 표시명, 보유 시작일을 저장합니다.
- 매수/매도 버튼은 예수금을 함께 차감/가산하는 거래 입력으로 동작합니다.
- 매수 시 예수금이 부족하면 음수 예수금으로 진행하지 않고 안내 메시지를 표시합니다.
- 선택 삭제는 현재 선택한 보유종목 1개를 삭제하는 삭제 버튼으로 정리했습니다.

3. 보유 시작일과 평단가 통화 개선
- 보유 시작일 입력 안내를 짧은 yyyymmdd 형식으로 정리했습니다.
- 보유 시작일을 비워두고 저장/수정하면 시스템 날짜를 자동 저장합니다.
- 미국 종목 평단가는 기본 KRW이며 USD/KRW 중 선택할 수 있습니다.
- 국내 종목은 평단가 통화를 KRW로 고정합니다.

4. 수익 탭 개선
- 매도 시 실현수익이 자동 기록됩니다.
- 수동 추가 시 상단 보유종목 드롭다운에서 고른 종목으로 행을 생성합니다.
- 매수단가, 매도단가, 1주당 수익, 총수익, 수익률 중심으로 표시합니다.
- 미국 종목 실현수익은 USD와 원화 환산액을 함께 표시합니다.

5. 배당 탭 개선
- 수익 탭과 배당 탭을 분리했습니다.
- 신규 배당만과 범위 불러오기 버튼을 분리했습니다.
- 배당 조회 범위는 보유 시작일 이후, 최근 1년, 최근 3년, 직접 날짜 지정, 전체 중 선택할 수 있습니다.
- 전체 범위에서는 날짜 입력칸을 숨기고, 최근 1년/최근 3년 선택 시 날짜를 자동 입력합니다.
- 배당 조회 대상은 전체 보유종목 또는 특정 보유종목으로 선택할 수 있습니다.
- 수동 추가도 선택한 보유종목을 기준으로 행을 생성합니다.
- 배당 중복 기준에서 보유 주수는 제외합니다.
- USD/KRW 환율이 없을 때 기존 원화 환산 배당액을 0으로 덮어쓰지 않도록 방어했습니다.

6. 종목명 데이터 업데이트 UX 보강
- v1.2에서 분리한 종목명 매핑 구조 위에 중앙 데이터팩 URL과 업데이트 버튼을 추가했습니다.
- 사용자 보정 데이터, 중앙 데이터, 앱 내장 데이터의 우선순위를 앱 동작 기준으로 정리했습니다.
- 사용자가 직접 입력한 표시명이 중앙 데이터 업데이트로 덮이지 않도록 사용자 보정 우선 구조를 유지합니다.
- 미매핑 종목은 AppData의 unmapped_symbols.json에 기록해 이후 데이터팩 보완에 활용할 수 있게 했습니다.

7. 종목 입력과 선택 동작 수정
- 종목명 데이터 재로드 후 기본 표시명 사전이 사라져 자동완성이 비어 보일 수 있던 문제를 수정했습니다.
- 국내 6자리 코드 입력 시 현재 로딩된 종목명 데이터의 .KS/.KQ/.KN 매핑을 먼저 적용하도록 수정했습니다.
- RKLB의 기본 한국어 표시명을 로켓랩으로 보완했습니다.
- 포트폴리오 보유종목 목록에서 선택된 행을 다시 클릭하면 선택이 해제되도록 변경했습니다.

8. 빌드 안정성 수정
- 실행 중인 portable 앱이 dist 폴더를 잠근 상태에서도 설치본은 별도 staging 빌드 결과를 사용하도록 변경했습니다.
- PyInstaller 또는 Inno Setup 실패를 설치 빌드 성공으로 오인하지 않도록 실패 시 즉시 중단하게 수정했습니다.
